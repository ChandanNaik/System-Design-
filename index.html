Grokking System Design Interview

Source: educative

Interview Process

  • Scope the problem
    1. Requirements clarifications
    2. System interface definition
    3. Don’t make assumptions.
    4. Ask clarifying questions to understand the constraints and use cases.
  • Sketch up an abstract design
    1. Back-of-the-envelope estimation
    2. Defining data model
    3. High-level design
    4. Building blocks of the system
    5. Relationships between them
  • Identify and address the bottlenecks
    1. Detailed design
    2. Identifying and resolving bottlenecks
    3. Use the fundamental principles of scalable system design
